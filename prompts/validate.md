# Validate 操作提示词

在 Ingest 完成后执行，用于生成 Obsidian Canvas 画布，在 Obsidian 中可视化验证本次知识提取结果。

---

## 标准 Validate（Ingest 后执行）

```
请根据刚才 Ingest 的结果，生成一个 Obsidian Canvas 文件用于可视化验证。

源文件：[填入刚才 Ingest 的文件路径，例如 sources/proposals/001_湛江电信升级规划方案.pdf]
今天日期：[填入当前日期，例如 2025-08-01]

Canvas 文件保存路径：wiki/canvases/[日期]_[源文件名].canvas

Canvas 布局规则：
1. 左侧：源文件节点（type: file，红色 color:"1"，指向 sources/ 实际路径）
2. 中部左列：本次【新建】的 wiki 页面（绿色分组 color:"4"，每页一个 file 节点）
3. 中部右列：本次【更新】的 wiki 页面（蓝色分组 color:"5"，每页一个 file 节点）
4. 右侧：被本次页面引用的【已有】wiki 页面（灰色 color:"6"，仅参考展示，不加分组）
5. 连线规则：
   - 源文件 → 每个新建/更新页面（label 写"新建"或"更新"）
   - 新建/更新页面之间若有 wikilink 引用关系 → 加连线（label 写引用内容摘要，15字以内）
   - 新建/更新页面 → 已有页面（label 写"引用"）

请严格按照 JSON Canvas 格式输出完整 .canvas 文件内容，然后将其写入上述路径。

写入后，输出简要摘要：
- 本次新建页面：[列表]
- 本次更新页面：[列表]
- Canvas 文件路径：[路径]
- 提示：在 Obsidian 中打开该 .canvas 文件进行人工确认
```

---

## 人工验证清单（在 Obsidian 打开 Canvas 后核对）

打开 Canvas 后，逐项确认：

- [ ] 源文件节点显示正确（文件名、路径无误）
- [ ] 新建页面数量与实际一致
- [ ] 更新页面数量与实际一致
- [ ] 页面间连线关系符合逻辑（引用关系是否合理）
- [ ] 没有遗漏应该新建但未新建的页面类型
- [ ] 行业/模块/阶段分类标注正确

若发现问题，直接告知 LLM 修正，例如：
```
Canvas 中发现：[具体问题描述]
请修正 wiki/[页面路径] 并重新生成 Canvas。
```

---

## 快速 Validate（仅生成摘要，不生成 Canvas）

```
请对刚才 Ingest 的结果做一次快速验证摘要，不生成 Canvas 文件：

输出格式：
## Ingest 验证摘要 — [源文件名]
- 识别行业：
- 识别方案阶段：
- 识别展厅类型：
- 识别客户层级：
- 新建页面（N个）：[列表]
- 更新页面（N个）：[列表]
- 发现政策引用（N条）：[列表，含政策名和时效]
- 发现资质需求（N项）：[列表]
- 潜在问题或待确认项：[如无则填"无"]
```
